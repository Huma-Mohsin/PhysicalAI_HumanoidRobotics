# Hardware Requirements

This course is technically demanding. It sits at the intersection of three heavy computational loads:
- **Physics Simulation** (Isaac Sim/Gazebo)
- **Visual Perception** (SLAM/Computer Vision)
- **Generative AI** (LLMs/VLA)

Because the capstone involves a "Simulated Humanoid," the primary investment must be in **High-Performance Workstations**. However, to fulfill the "Physical AI" promise, you also need **Edge Computing Kits** (brains without bodies) or specific robot hardware.

## 1. The "Digital Twin" Workstation (Required per Student)

This is the **most critical component**. NVIDIA Isaac Sim is an Omniverse application that requires "RTX" (Ray Tracing) capabilities. Standard laptops (MacBooks or non-RTX Windows machines) will **not work**.

| Component | Specification | Rationale |
|-----------|--------------|-----------|
| **GPU** (The Bottleneck) | **NVIDIA RTX 4070 Ti (12GB VRAM)** or higher | You need high VRAM to load the USD (Universal Scene Description) assets for the robot and environment, plus run the VLA (Vision-Language-Action) models simultaneously. |
| **GPU** (Ideal) | **RTX 3090 or 4090 (24GB VRAM)** | Allows for smoother "Sim-to-Real" training with larger scene complexity. |
| **CPU** | Intel Core i7 (13th Gen+) or AMD Ryzen 9 | Physics calculations (Rigid Body Dynamics) in Gazebo/Isaac are CPU-intensive. |
| **RAM** | **64 GB DDR5** (32 GB absolute minimum) | 32 GB will crash during complex scene rendering. 64 GB provides headroom. |
| **OS** | **Ubuntu 22.04 LTS** | While Isaac Sim runs on Windows, ROS 2 (Humble/Iron) is native to Linux. Dual-booting or dedicated Linux machines are mandatory for a friction-free experience. |

:::warning GPU Requirement
Standard laptops (MacBooks or non-RTX Windows machines) **cannot run** NVIDIA Isaac Sim. You must have an NVIDIA RTX GPU.
:::

## 3. The Robot Lab

For the "Physical" part of the course, you have three tiers of options depending on budget.

### Option A: The "Proxy" Approach (Recommended for Budget)

Use a quadruped (dog) or a robotic arm as a proxy. The software principles (ROS 2, VSLAM, Isaac Sim) transfer 90% effectively to humanoids.

| Robot | Price | Pros | Cons |
|-------|-------|------|------|
| **Unitree Go2 Edu** | $1,800 - $3,000 | Highly durable, excellent ROS 2 support, affordable enough to have multiple units | Not a biped (humanoid) |

### Option B: The "Miniature Humanoid" Approach

Small, table-top humanoids.

| Robot | Price | Notes |
|-------|-------|-------|
| **Unitree G1** | ~$16,000 | Research-grade humanoid with open SDK |
| **Robotis OP3** | ~$12,000 | Older but stable platform |
| **Hiwonder TonyPi Pro** | ~$600 | Budget alternative (runs on Raspberry Pi, limited Isaac ROS support) |

:::warning Budget Humanoids
Cheap kits (Hiwonder) usually run on Raspberry Pi, which **cannot run NVIDIA Isaac ROS efficiently**. Use these only for kinematics (walking) and use the Jetson kits for AI.
:::

### Option C: The "Premium" Lab (Sim-to-Real specific)

If the goal is to actually deploy the Capstone to a real humanoid:

| Robot | Price | Why? |
|-------|-------|------|
| **Unitree G1 Humanoid** | ~$16,000 | One of the few commercially available humanoids that can walk dynamically and has an SDK open enough for students to inject their own ROS 2 controllers. |

## 4. Summary of Architecture

To teach this successfully, your lab infrastructure should look like this:

| Component | Hardware | Function |
|-----------|----------|----------|
| **Sim Rig** | PC with RTX 4080 + Ubuntu 22.04 | Runs Isaac Sim, Gazebo, Unity, and trains LLM/VLA models |
| **Edge Brain** | Jetson Orin Nano | Runs the "Inference" stack. Students deploy their code here |
| **Sensors** | RealSense Camera + Lidar | Connected to the Jetson to feed real-world data to the AI |
| **Actuator** | Unitree Go2 or G1 (Shared) | Receives motor commands from the Jetson |

:::danger Critical Note
If you do not have access to RTX-enabled workstations, we must restructure the course to rely entirely on **cloud-based instances** (like AWS RoboMaker or NVIDIA's cloud delivery for Omniverse), though this introduces significant latency and cost complexity.
:::

### 2. Local "Bridge" Hardware

You cannot eliminate hardware entirely for "Physical AI." You still need the edge devices to deploy the code physically.

- **Edge AI Kits**: Jetson Kit ($700 one-time purchase)
- **Robot**: One physical robot for final demo ($3,000 - Unitree Go2)

### 3. The Latency Trap (Hidden Cost)

:::warning Cloud Limitations
Simulating in the cloud works well, but **controlling a real robot from a cloud instance is dangerous** due to latency.

**Solution**: Students train in the Cloud, download the model (weights), and flash it to the local Jetson kit.
:::
